## 📋 AICE 모의고사 플랫폼 요구사항 정의서 (Draft)

### 1. 모바일 인트로 페이지 (Mobile Intro)

- **목적**: 모바일 접속자에게 PC 환경 접속 유도 및 공유 기능 제공.
- **주요 요소**:
  - **상단**: AICE 로고 배치.
  - **중앙**: 도서 표지 이미지 및 안내 문구 ("PC 실습 환경에 최적화된 서비스입니다").
  - **하단 액션**:
    - `카카오톡으로 링크 보내기` (Orange Button): 카카오톡 공유 API 연동.
    - `URL 복사` (Ghost Button): 클립보드 복사 기능.

### 2. PC 도서 인증 페이지 (Auth)

- **목적**: 구매 도서를 활용한 사용자 인증.
- **주요 요소**:
  - **인증 폼**: 카드 타입의 중앙 정렬 레이아웃.
  - **입력 필드**: 8자리 난수 입력 (각 숫자별 개별 Input Box 형태 UI).
  - **안내 문구**: "도서 [페이지 21] 하단에 인쇄된 8자리 난수를 입력해 주세요."
  - **기능**:
    - `인증하기` 버튼: 유효성 검사 및 세션 생성.
    - `이 브라우저에서 인증 상태 유지` 체크박스: 쿠키/로컬 스토리지를 이용한 자동 로그인 유지.

### 3. PC 모의고사 로비 페이지 (Lobby)

- **목적**: 회차별 모의고사 선택 및 실습/학습 진입.
- **주요 요소**:
  - **GNB**: 로고 및 통합 검색창 ("허용 라이브러리 공식 문서 검색" 기능).
  - **콘텐츠 영역**: 2컬럼 그리드 형태의 모의고사 카드.
    - **카드 구성**: 회차 제목, 요약 설명, `Google Colab 실습` (메인 액션), `정답 및 해설 보기`, `해설 강의 수강`.
  - **사이드바 (Right)**:
    - **회차 바로가기**: 1회차 ~ 8회차 퀵 네비게이션.
    - **내 점수 기록**: Local Storage에 저장된 사용자 학습 이력 표시.
