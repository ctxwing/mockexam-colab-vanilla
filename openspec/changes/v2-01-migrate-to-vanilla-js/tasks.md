# Tasks.md 작성 및 관리 가이드

## 완료 내역 관리 규정

### Phase (N) 레벨 완료 시
각 Phase(대분류 태스크: 1, 2, 3...)가 완료되면 다음 절차를 순차적으로 수행한다:

1. **완료 내역 문서 작성**: `3_prj_docs/완료내역/TASK_<X>_완료내역.md` 파일 생성
   - 파일 내부에 작성일자 기록 (형식: YYYY-MM-DD)
   - 완료된 작업 내용 요약 기록
2. **체크리스트 업데이트**: 해당 Phase의 모든 하위 태스크를 `[x]`로 변경
3. **Git 커밋 및 푸시**:
   - 커밋 메시지는 한글로 작성
   - 개행 문자 활용하여 한 줄로 표시하지 않음

### Section (N.N) 레벨 완료 시
각 섹션(중분류 태스크: 1.1, 1.2, 2.1, 2.2...)이 완료되면 다음 절차를 수행한다:

1. **완료 내역 문서 작성**: `3_prj_docs/완료내역/TASK_<X>_<NN>_완료내역.md` 파일 생성
   - 예: `3_prj_docs/완료내역/TASK_1_1_완료내역.md` (1.1 디렉토리 구조 생성 완료)
   - 파일 내부에 작성일자 기록 (형식: YYYY-MM-DD)
   - 완료된 작업 내용 요약 기록
2. **체크리스트 업데이트**: 해당 섹션의 모든 하위 태스크를 `[x]`로 변경
3. **Git 커밋 및 푸시**:
   - 커밋 메시지는 한글로 작성
   - 개행 문자 활용하여 한 줄로 표시하지 않음

### Section (N.N) 레벨 완료 시
각 섹션(중분류 태스크: 1.1, 1.2, 2.1, 2.2...)이 완료되면 다음 절차를 수행한다:

   - 예: `3_prj_docs/완료내역/TASK_1_1_완료내역.md` (1.1 디렉토리 구조 생성 완료)
   - 파일 내부에 작성일자 기록 (형식: YYYY-MM-DD)
   - 완료된 작업 내용 요약 기록
2. **체크리스트 업데이트**: 해당 섹션의 모든 하위 태스크를 `[x]`로 변경
3. **Git 커밋 및 푸시**:
   - 커밋 메시지는 한글로 작성
   - 개행 문자 활용하여 한 줄로 표시하지 않음
   - 예시:
     ```
     feat: 1.1 디렉토리 구조 생성 완료

     - prj_source 루트 디렉토리 생성
     ```
