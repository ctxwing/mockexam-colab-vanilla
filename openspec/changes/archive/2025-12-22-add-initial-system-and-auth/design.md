# 디자인: 초기 시스템 아키텍처 및 인증 설계

## 맥락
본 시스템은 독자가 도서 구매 후 제공되는 난수 코드를 입력하여 Google Colab 실습 환경으로 이동하는 '게이트웨이' 역할을 합니다. 높은 트래픽보다는 저비용, 저유지보수, 그리고 깔끔한 UI/UX가 핵심입니다.

## 목표 / 비목표
- **목표**: 
  - Next.js App Router 기반의 모던한 프로젝트 구조 구축
  - 모바일 사용자를 위한 PC 접속 안내 UX 제공
  - DB 서버나 클라이언트 DB(pglite) 없이 완전하게 작동하는 Stateless 인증 로직
  - shadcn/ui를 활용한 프리미엄 UI 디자인 적용
- **비목표**:
  - 사용자 개별 계정(ID/PW) 관리 기능
  - 자동 채점 및 결과 저장 시스템
  - 유료 DB 서비스 도입

## 기술적 결정
- **프레임워크 버전**: `next@16.x` 및 `react@19.x`를 기본으로 하되, `3_prj_docs/NextJS_React_보안취약점패치적용_보고_참고.md`를 참고하여 보안 수준을 유지합니다. (문서상의 v16은 향후 릴리즈 모델로 간주하고 최신 안정화 버전 적용)
- **DB-less Architecture**: 고객의 강력한 요구사항에 따라 `pglite` 및 `Drizzle ORM`을 배제하고, 인증 코드는 서버 측 보호되는 환경 변수(`.env`) 또는 상수로 관리합니다. 이는 Vercel 배포 시 유지비 0원을 보장합니다.
- **UI 라이브러리**: `shadcn/ui`와 `Tailwind CSS`를 사용하여 `ctx.plan.md`의 "미려한 UI" 요구사항을 충족합니다.
- **반응형 대응**: `User-Agent` 검사 또는 클라이언트 사이드 화면 너비 체크를 통해 모바일 기기 접속 시 전용 안내 모달/페이지를 노출합니다.

- **영속성**: 별도의 영속성 데이터(사용자 정보 등)를 저장하지 않으며, 매 접속 시 인증 코드를 재확인하는 정적/서버리스 방식을 유지합니다.
