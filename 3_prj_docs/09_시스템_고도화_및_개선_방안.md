# 시스템 고도화 및 배포 개선 방안 (Future Roadmap)

현재 시스템의 관리 편의성을 극대화하기 위한 향후 개선 및 고도화 방안을 기록합니다.

## 1. 콘텐츠 통합 관리 환경 개선 (Public 연동)

### 현황
- 현재는 해설지 텍스트와 시험 구성 정보가 Apps Script 소스 코드 내부에 하드코딩되어 배포됨.

### 개선 방안
- **Public 폴더 동기화**: 웹사이트 소스 코드의 `/public/solutions/` 등에 MD 파일이나 JSON 설정 파일을 두면, Apps Script가 이 URL을 직접 호출하여 내용을 읽어오게 합니다.
- **장점**: 개발자가 웹 소스만 커밋(Push)하면, 고객이 '1번 초기화' 버튼을 눌렀을 때 최신 콘텐츠가 고객의 드라이브에 자동으로 세팅됩니다. (소스 코드 수정 불필요)

## 2. 스마트 매칭 엔진 (2번 메뉴 고도화)

### 현황
- 현재 2번 메뉴는 드롭다운 목록만 갱신하며, 실제 파일 연결은 사용자가 수동으로 선택해야 함.

### 개선 방안
- **자동 파일 ID 매핑**: 특정 명명 규칙(예: `시험ID_해설`, `시험ID_폼`)을 기반으로, 폴더 내를 검색하여 `exams` 시트의 `solutionDocId` 등을 한 번에 채워주는 기능을 추가합니다.
- **장점**: 수십 개의 시험을 관리할 때 수작업 실수를 0%로 줄일 수 있습니다.

## 3. 인증 시스템 고도화

### 현황
- 20개의 정적 코드를 단순 비교하는 방식.

### 개선 방안
- **일회용 코드 권한**: 사용자가 코드를 사용하면 특정 기기에 귀속되거나 사용 횟수를 차단하는 로직을 구글 시트의 `note` 컬럼 등에 기록하여 보안을 강화할 수 있습니다.

## 4. Colab 주소 자동 생성 (하드코딩 제거)

### 현황
- `exams` 시트의 `colabUrl`을 수동으로 관리.

### 개선 방안
- **URL 템플릿 도입**: `settings` 시트의 사이트 주소를 기반으로 `${siteUrl}/colab/${id}.ipynb` 형식으로 자동 계산되게 합니다.
- **장점**: 도메인이 바뀌더라도 데이터 수정 없이 모든 링크가 일괄 정상 작동합니다.
