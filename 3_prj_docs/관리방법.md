# 관리 방법

## OTP 인증 코드 관리

### 개요

이 프로젝트는 GitHub Pages 정적 호스팅을 사용하므로, 데이터베이스 없이 파일 기반으로 OTP를 관리합니다. 보안을 위해 **SHA-256 단방향 해시**를 사용합니다.

### 파일 구조

```
prj_source/
├── data/
│   ├── otp-codes.json      # 원본 OTP 코드 (개발용, .gitignore 관리)
│   └── otp-list.json       # SHA-256 해시 목록 (배포용)
├── tools/
│   ├── admin.html          # 관리자 로그인 페이지
│   └── generate-otp-hashes.html  # 해시 생성 도구
└── js/
    └── auth.js             # OTP 검증 로직
```

## OTP 코드 추가/삭제/변경

### 방법 1: 관리자 도구 사용 (권장)

#### 1단계: 관리자 로그인

1. **개발 서버 실행**
   ```bash
   ./start.sh
   ```

2. **관리자 페이지 접속**
   - URL: http://localhost:53500/tools/admin.html
   - 기본 비밀번호: `admin123`

#### 2단계: 해시 생성

1. 로그인 후 자동으로 해시 생성 도구로 이동
2. JSON 형식으로 OTP 코드 입력:
   ```json
   {
     "codes": [
       "AICE2025",
       "AICE2026",
       "NEWCODE01"
     ]
   }
   ```
3. "해시 생성" 버튼 클릭
4. 결과 복사

#### 3단계: 배포 파일 업데이트

1. `prj_source/data/otp-list.json` 파일 열기
2. 복사한 해시로 교체
3. 테스트: http://localhost:53500/pages/auth.html

#### 4단계: 배포

```bash
git add prj_source/data/otp-list.json
git commit -m "Update OTP codes"
git push origin main
./deploy-ghpages.sh
```

## 보안 관리

### 관리자 비밀번호 변경

기본 비밀번호 `admin123`를 변경하려면:

1. **새 비밀번호의 SHA-256 해시 생성**

   브라우저 콘솔에서:
   ```javascript
   async function sha256(text) {
       const encoder = new TextEncoder();
       const data = encoder.encode(text);
       const hashBuffer = await crypto.subtle.digest('SHA-256', data);
       const hashArray = Array.from(new Uint8Array(hashBuffer));
       return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
   }
   sha256('새비밀번호').then(console.log);
   ```

2. **`prj_source/tools/admin.html` 수정**
   ```javascript
   const PASSWORD_HASH = "새로운해시값";
   ```

3. **커밋 및 배포**

### .gitignore 관리

# prj_source/.gitignore

## 트러블슈팅

### Q: 배포 후 OTP 인증이 안 됩니다

**A:**
- `otp-list.json`이 정상적으로 배포되었는지 확인
- 브라우저 개발자 도구 → 네트워크 탭에서 `otp-list.json` 로그 확인
- 해시 파일 형식 확인:
  ```json
  {
    "hashes": ["sha256hash1", "sha256hash2", ...]
  }
  ```

### Q: 해시 생성이 복잡합니다

**A:** 관리자 도구를 사용하면 브라우저에서 바로 생성 가능합니다. 자주 변경해야 한다면 즐겨찾기에 추가하세요.

### Q: 원본 코드가 노출될까요?

**A:**
- SHA-256 단방향 해시 사용으로 역공매 매우 어려움
- `otp-codes.json`은 `.gitignore`로 Git에 저장되지 않음
- 완전한 보안이 필요하면 별도 인증 서버 구축 권장

### Q: 관리자 비밀번호를 잊어버렸습니다

**A:**
1. GitHub 저장소에서 `admin.html`의 `PASSWORD_HASH` 확인
2. 기본값: `240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9` (admin123)
3. 또는 새 해시 생성 후 파일 수정
