# 구글 폼 인증 코드 수정 방지 및 자동 제출 운영 가이드

이 문서는 모의고사 답안 제출용 구글 폼에서 '도서 인증 코드'가 사용자에게 노출되지 않거나, 노출되더라도 수정을 원천 봉쇄하기 위한 운영 전략을 기록합니다.

## 1. 배경 및 문제점
- **현상**: Next.js 대시보드에서 `entry.xxxx` 인자를 통해 구글 폼의 인증 코드 항목을 자동으로 채워주지만, 구글 폼 화면상에서 사용자가 이 값을 임의로 수정하거나 삭제할 수 있음.
- **제약**: 구글 폼은 공식적으로 '읽기 전용(Read-only)' 필드나 '숨김(Hidden)' 필드를 지원하지 않음.

## 2. 권장 솔루션: 섹션 건너뛰기(Section Skipping) 전략

구글 폼의 섹션 이동 로직을 활용하여, 인증 코드 입력 칸을 물리적으로 숨기면서 데이터는 전송하는 방식입니다.

### 2.1 구현 단계
1.  **필드 이동**: '인증 코드' 질문 항목을 설문지의 **맨 마지막** 위치로 이동시킵니다.
2.  **섹션 추가**: 인증 코드 질문 바로 위에 **[섹션 추가]**를 클릭하여 독립된 섹션(예: 섹션 2)으로 분리합니다.
3.  **제출 로직 설정**: 이전 섹션(실제 문제가 포함된 섹션 1)의 하단 설정에서 **"섹션 1 다음에 진행할 작업"**을 **[설문지 제출]**로 변경합니다.
4.  **필수 옵션 해제**: 인증 코드 질문의 **'필수(Required)'** 설정을 반드시 **해제**해야 합니다. (필수 설정 시 내용을 확인하지 않으면 제출되지 않음)

### 2.2 기대 효과
- **완벽한 숨김**: 사용자는 답안 작성 후 바로 제출 버튼을 보게 되며, 인증 코드 입력 섹션은 브라우저에 렌더링되지 않습니다.
- **수정 봉쇄**: 화면에 나타나지 않으므로 사용자가 값을 수정할 방법이 없습니다.
- **백그라운드 전송**: URL의 `entry.xxxx`를 통해 전달된 값은 섹션을 건너뛰더라도 구글 폼의 제출 데이터에 포함되어 구글 시트로 기록됩니다.
- **유지보수 용이**: 시험 질문이 수시로 변경되더라도 섹션 1만 수정하면 되므로 운영 효율성이 높습니다.

## 3. 대안: 응답 확인(Validation) 활용

만약 인증 코드를 화면에 보여주어야만 하는 상황이라면 이 방식을 사용합니다.

- **설정**: 질문 항목의 '응답 확인' -> '정규표현식' 선택
- **패턴**: `^[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}$` (필요한 형식에 맞춰 조정)
- **메시지**: "인증 코드가 수정되었습니다. 대시보드에서 다시 접속해주세요."
- **효과**: 사용자가 값을 한 글자라도 수정하면 경고 메시지와 함께 제출이 차단됩니다.

## 4. 향후 고려사항 (고도화 단계)
질문 항목의 변경이 잦지 않고 프리미엄 UI가 필요한 경우, 구글 폼을 쓰지 않고 Next.js 내부에 전용 제출 인터페이스를 구축하여 `input type="hidden"`을 사용하는 방식을 고려할 수 있습니다.
