# 시스템 운영 및 관리 가이드
작성일자: 2025-12-22

본 문서는 교육용 가상 모의고사 시스템의 유지보수 및 운영을 위한 가이드라인입니다.

## 1. 인증 코드 관리 방법
사용자 인증 코드는 `lib/auth-codes.ts`에 상수로 정의되어 있거나, 배포 환경(Vercel)의 환경 변수로 관리됩니다.

### 1.1 Vercel 환경 변수 설정 (권장)
운영 중 코드를 동적으로 변경하려면 Vercel Dashboard에서 다음 환경 변수를 설정하세요.
- **Key**: `AUTH_CODES_JSON`
- **Value**: `["CODE1-XXXX-XXXX-XXXX", "CODE2-XXXX-XXXX-XXXX"]` (JSON 배열 형식)

### 1.2 소스 코드 수정
`lib/auth-codes.ts` 파일의 `AUTH_CODES` 배열을 직접 수정하여 코드를 추가하거나 삭제할 수 있습니다. 수정 후에는 다시 배포가 필요합니다.

## 2. 모의고사 콘텐츠 수정 (Colab, 구글 폼, 데이터 URL)
모의고사 목록 및 각 항목의 링크는 `lib/exam-data.ts`에서 관리합니다.

```typescript
export const EXAMS: Exam[] = [
  {
    id: "exam-unique-id",
    title: "시험 제목",
    description: "시험 상세 설명",
    colabUrl: "Google Colab 링크",
    formUrl: "답안 제출용 구글 폼 링크",
    dataUrl: "실습용 CSV 데이터 URL",
    solutionUrl: "해설 페이지 경로 (기본: /dashboard/solution/[id])"
  },
  // ... 추가
];
```

## 3. 모바일 접속 안내 수정
모바일 기기 접속 시 노출되는 안내 문구 및 로직은 다음 파일에서 수정 가능합니다.
- **안내 문구**: `components/device-warning.tsx`
- **감지 로직**: `app/auth/page.tsx` 내 `useEffect` 부분

## 4. 정답 및 해설 페이지 관리
해설 페이지는 `app/dashboard/solution/[id]/page.tsx`를 통해 동적으로 렌더링됩니다. 현재는 준비 중 화면으로 구성되어 있으며, 실제 콘텐츠는 해당 폴더 내에 별도의 컴포넌트를 작성하거나 정적 HTML을 삽입하여 보충할 수 있습니다.
