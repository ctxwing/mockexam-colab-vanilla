# 최적 운용 방안 후보 검토 비교표

작성일: 2025-12-23
프로젝트: Colab 기반 교육용 가상 모의고사 웹 서비스

---

## 1. 검토 배경

고객 요구사항의 핵심 제약조건:
- **DB 사용 불가**: 유지보수 비용 최소화를 위해 DB를 사용하지 않는 구조
- **간편 운영**: 고객이 GitHub을 다룰 수 없음
- **정적 호스팅**: Vercel, Netlify 등 서버리스 환경

관리 대상 데이터:
1. 모의고사 메타데이터 (목록, 링크, 상태)
2. 모의고사 해설 콘텐츠 (마크다운/문서)
3. 인증 코드 (20개)
4. 답안 제출 결과 (Google Forms)

## 2. 평가 지표 (5대 지표)

| 지표 | 설명 | 가중치 |
|------|------|--------|
| **① 관리자 편의성** | 고객이 직접 데이터를 수정할 수 있는 용이성 | 25% |
| **② 구현 복잡도** | 개발 및 연동에 필요한 기술적 난이도 | 20% |
| **③ 유지보수 비용** | 운영 시 발생하는 비용 및 개발자 개입 빈도 | 20% |
| **④ 안정성/신뢰성** | 시스템 장애 가능성 및 데이터 손실 위험 | 20% |
| **⑤ 확장성** | 향후 기능 추가 및 데이터 증가 대응 가능성 | 15% |

## 3. 후보 방안

### 방안 A: 현행 유지 (정적 파일)
- 메타데이터: `lib/exam-data.ts` (TypeScript)
- 해설: `public/solutions/*.md` (Markdown)
- 인증코드: 환경변수 (AUTH_CODES_JSON)
- 변경 시 개발자가 Git commit → 재배포

### 방안 B: Google Sheets 단독
- 메타데이터: Google Sheets
- 해설: Google Sheets 셀에 마크다운 저장
- 인증코드: Google Sheets
- GAS(Apps Script)로 데이터 제공

### 방안 C: Google Sheets + Notion
- 메타데이터: Google Sheets
- 해설: Notion 페이지
- 인증코드: Google Sheets
- 이중 관리 필요

### 방안 D: Google Drive Folder (GDF) 통합
- 메타데이터: Google Sheets (관리 시트)
- 해설: Google Docs (같은 폴더 내)
- 답안 제출: Google Forms (같은 폴더 내)
- 인증코드: Google Sheets
- GAS로 자동화 (드롭다운 연동)

### 방안 E: Headless CMS (Contentful, Sanity)
- 외부 CMS 서비스 사용
- GUI 기반 콘텐츠 관리
- API 연동 필요

## 6. 최종 결정

### 🏆 선정: 방안 D - Google Drive Folder (GDF) 통합

**핵심 선정 사유:**
1. 고객이 이미 Google Colab/Forms 사용 → 동일 생태계
2. Sheets 드롭다운으로 Docs/Forms 연동 → 오타/오류 방지
3. 해설 편집 시 Google Docs 에디터 활용 → 편리한 UX
4. 모든 데이터가 하나의 GDF 안에서 통합 관리
5. 운영 가이드 문서화 용이

**예상 리스크 및 대응:**
- Google 계정 의존: 팀 드라이브 사용 권장
- API 할당량: 변경 빈도 낮음 (월 1-2회) → 문제 없음
- 데이터 동기화: 파일 ID 기반 참조로 안정성 확보
