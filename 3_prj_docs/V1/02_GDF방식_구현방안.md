# Google Drive Folder (GDF) 방식 구현 방안

작성일: 2025-12-23
프로젝트: Colab 기반 교육용 가상 모의고사 웹 서비스

---

## 1. 결정 사유

### 1.1 선정 배경
고객 요구사항 분석 결과:
- **DB 미사용** 제약 하에서 콘텐츠 관리 방안 필요
- 고객이 **GitHub 사용 불가** → 개발자 개입 없이 운영 가능해야 함
- 고객이 **Google Colab, Forms 사용 중** → Google 생태계 친숙

### 1.2 GDF 방식 채택 근거

| 요소 | GDF 방식의 장점 |
|------|----------------|
| 관리 통합 | 모든 데이터가 하나의 폴더 안에서 관리 |
| 편의성 | Sheets 드롭다운으로 Docs/Forms 선택 (오타 방지) |
| 편집 UX | Google Docs 에디터로 해설 편집 (마크다운 변환 불필요) |
| 권한 관리 | 폴더 단위 공유 설정으로 간편 |
| 무료 비용 | Google Workspace 무료 티어로 충분 |

### 1.3 대안 대비 우위

| 항목 | 현행(정적 파일) | GDF 방식 |
|------|----------------|----------|
| 고객 수정 가능 | ❌ | ✅ |
| 재배포 필요 | 매번 필요 | 불필요 (캐싱 갱신만) |
| 해설 편집 | 마크다운 직접 편집 | Google Docs 에디터 |
| 학습 비용 | - | Google Sheets/Docs (이미 익숙) |

## 2. 폴더 구조

```
📁 GDF (Google Drive Folder)
│
├── 📊 문항관리.gsheet (Management Sheet)
│   ├── [시트] exams: 모의고사 목록
│   ├── [시트] auth_codes: 인증 코드
│   └── [시트] settings: 설정값 (폴더 ID 등)
│
├── 📁 Solutions/
│   ├── 빅분기_1회_해설.gdoc
│   ├── 빅분기_2회_해설.gdoc
│   ├── AICE_1회_해설.gdoc
│   └── AICE_2회_해설.gdoc
│
├── 📁 Forms/
│   ├── 빅분기_1회_제출폼.gform
│   ├── 빅분기_2회_제출폼.gform
│   └── ...
│
├── 📁 Colabs/
│   └── (Colab 노트북 링크 또는 바로가기)
│
└── 📁 Responses/
    └── (Forms 응답 자동 저장 시트)
```

## 3. 종합 관리 시트 (S) 설계

### 3.1 시트 구조

#### [시트 1] exams - 모의고사 목록

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | 텍스트 | 고유 식별자 | bigdata-01 |
| title | 텍스트 | 표시 제목 | 빅데이터 분석기사 1회 |
| description | 텍스트 | 간단 설명 | 제7회 기출 복원 |
| category | 드롭다운 | 분류 | bigdata / aice |
| solutionDoc | 드롭다운 | 해설 문서 (Solutions/ 폴더에서 선택) | 빅분기_1회_해설 |
| solutionDocId | 자동 | 선택된 문서의 ID | 1abc2def3... |
| solutionUrl | 자동 | 문서 공개 URL | https://docs.google.com/... |
| formName | 드롭다운 | 제출 폼 (Forms/ 폴더에서 선택) | 빅분기_1회_제출폼 |
| formUrl | 자동 | 폼 URL | https://docs.google.com/forms/... |
| colabUrl | 텍스트 | Colab 노트북 URL | https://colab.research.google.com/... |
| dataUrl | 텍스트 | 실습 데이터 URL (GitHub Raw 등) | https://raw.githubusercontent.com/... |
| order | 숫자 | 표시 순서 | 1 |
| isActive | 체크박스 | 활성화 여부 | TRUE / FALSE |

#### [시트 2] auth_codes - 인증 코드

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| code | 텍스트 | 인증 코드 (16자리) | ABCD-EFGH-IJKL-MNOP |
| isActive | 체크박스 | 사용 가능 여부 | TRUE |
| note | 텍스트 | 메모 (선택) | 초판 1쇄 |

#### [시트 3] settings - 설정

| 컬럼 | 값 |
|------|-----|
| solutionsFolderId | (Solutions 폴더 ID) |
| formsFolderId | (Forms 폴더 ID) |
| lastUpdated | (마지막 수정일) |
| cacheKey | (캐시 무효화용 키) |

### 3.2 드롭다운 자동화 (GAS)

**동작 방식:**
1. 시트 열기(onOpen) 시 Solutions/Forms 폴더의 파일 목록 로드
2. solutionDoc, formName 컬럼에 드롭다운 옵션으로 설정
3. 선택 시(onEdit) 해당 파일의 ID와 URL 자동 입력

**장점:**
- 파일명 직접 입력 불필요 → 오타 방지
- 파일 추가 시 드롭다운에 자동 반영

## 4. 문항 답변 결과 Sheet 및 통계

### 4.1 Forms 응답 연동

Google Forms 제출 시 자동으로 응답 스프레드시트 생성됨.
→ `Responses/` 폴더에 저장되도록 설정

### 4.2 응답 시트 구조 (자동 생성)

| 타임스탬프 | 이메일 | Q1 답안 | Q2 답안 | ... | Q10 답안 | 파일 업로드 |
|-----------|--------|---------|---------|-----|---------|------------|
| 2025-12-23 10:00 | user@gmail.com | [A, B, C] | 12.34 | ... | 0.15 | (파일 링크) |

### 4.3 통계 시트 (선택 구현)

관리 시트에 통계 탭 추가 가능:

| 항목 | 산출 방식 |
|------|----------|
| 총 응답 수 | Forms 응답 행 수 |
| 회차별 응답 수 | COUNTIF로 집계 |
| 일별 응답 추이 | 타임스탬프 기준 집계 |

**자동 채점은 프로젝트 범위 외** (고객 요구사항 명시)

## 5. 해설 문서 (Google Docs)

### 5.1 문서 구조

각 모의고사별 하나의 Google Docs 파일:
- 제목, 개요
- 작업형 1유형 (Q1~Q3) - 데이터 전처리
- 작업형 2유형 (Q4) - 모델 구축
- 작업형 3유형 (Q5~Q10) - 통계 검정
- Forms 제출 가이드 (선택)

### 5.2 공유 설정

| 설정 | 값 |
|------|-----|
| 공개 범위 | "링크가 있는 모든 사용자" |
| 권한 | "뷰어" (읽기 전용) |

### 5.3 웹 앱 연동 방식

**옵션 A: iframe 임베드**
- Docs 미리보기 URL을 iframe으로 표시
- 장점: 구현 단순
- 단점: 스타일 커스텀 제한

**옵션 B: GAS 프록시 (권장)**
- GAS가 Docs 내용을 HTML로 변환하여 반환
- 장점: 비공개 문서도 접근 가능, 커스텀 렌더링
- 단점: GAS 개발 필요

**선택: 옵션 A (iframe) 1차 구현, 필요시 B로 전환**

## 7. GAS 웹앱 API 엔드포인트

### 7.1 엔드포인트 목록

| 엔드포인트 | 파라미터 | 반환 | 용도 |
|-----------|---------|------|------|
| `?action=getExams` | - | JSON 배열 | 모의고사 목록 조회 |
| `?action=validateCode&code=XXX` | code | boolean | 인증 코드 검증 |
| `?action=getSolution&id=XXX` | examId | HTML 문자열 | 해설 내용 조회 |

### 7.2 캐싱 전략

- Next.js ISR (revalidate: 86400) → 24시간 캐싱
- 관리자가 "배포" 버튼 클릭 시 → Vercel 재빌드 트리거
- settings 시트의 cacheKey 변경 → 캐시 무효화

## 8. 보안 고려사항

| 항목 | 대응 방안 |
|------|----------|
| Sheets 노출 | GAS 웹앱 통해서만 접근, 직접 URL 비공개 |
| 인증 코드 유출 | 대소문자 무시, 중복 검증 없음 (단순 인증) |
| Docs 무단 접근 | "링크가 있는 사용자"로 제한, URL 비공개 |
| GAS 오남용 | API 호출 빈도 제한 (토큰 추가 가능) |
