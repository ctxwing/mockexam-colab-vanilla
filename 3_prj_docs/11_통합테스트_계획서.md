# 통합 테스트 계획서

본 문서는 프로젝트의 품질 보증을 위한 E2E(End-to-End) 테스트 시나리오 및 기기별 반응형 테스트 계획을 설명합니다. 배포 관련 절차는 [Vercel 배포 절차](./Vercel_배포절차.md)를 참고하십시오.

---

## 🧪 1. E2E 테스트 계획 (End-to-End Test Plan)

사용자의 실제 이용 흐름을 처음부터 끝까지 검증하는 테스트 시나리오입니다.

### 1-1. 핵심 시나리오: "정상 학습 프로세스"
1. **인증 단계**:
    - `/auth` 페이지 접속 후 유효한 16자리 코드 입력.
    - 대소문자 혼합 및 하이픈(-) 위치에 관계없이 인증 성공 확인.
    - 성공 시 `/dashboard`로 자동 이동되는지 확인.
2. **대시보드 탐색**:
    - 모의고사 리스트가 정상적으로 로드되는지 확인 (GAS 연동 데이터).
    - '데이터 복사' 클릭 시 클립보드에 정확한 URL이 복사되는지 확인.
3. **실습 및 제출**:
    - 'Colab 접속' 클릭 시 새 탭으로 해당 노트북이 열리는지 확인.
    - '답안 제출' 클릭 시 구글 폼으로 이동하며, **인증 코드가 자동 입력**되어 있는지 확인.
4. **해설 열람**:
    - '모범답안 보기' 클릭 시 내부 뷰어에서 해설 내용이 렌더링되는지 확인.

### 1-2. 보안/예외 시나리오
1. **비인가 접근**: 인증 없이 `/dashboard` 직접 접속 시 `/auth`로 강제 리다이렉트되는지 확인.
2. **잘못된 코드**: 16자 미만 또는 잘못된 코드 입력 시 에러 메시지 노출 확인.
3. **Open Redirect 방지**: `/auth?redirect=http://abcd1234.com` 접속 후 인증 시 외부가 아닌 `/dashboard`로 이동되는지 확인.

## 📱 2. 반응형/기기 최적화 테스트 계획

다양한 기기 환경에서의 UX 일관성을 검증합니다.

### 2-1. 모바일 환경 (iPhone/Android 기준)
- **모바일 감지 팝업**: 스마트폰으로 접속 시 "PC 접속 권장" 팝업이 즉시 노출되는지 확인.
- **레이아웃 대응**: 모바일 웹뷰에서 대시보드 카드가 한 줄(Single Column)로 정렬되는지 확인.
- **주소 복사 기능**: 팝업 내 '링크 복사' 버튼 작동 확인.

### 2-2. 태블릿/데스크탑 환경
- **데스크탑 최적화**: 넓은 화면에서 2열 또는 3열 그리드 레이아웃 유지 확인.
- **다크 모드 지원**: 시스템 설정 또는 토글에 따라 다크/라이트 테마가 매끄럽게 전환되는지 확인.
