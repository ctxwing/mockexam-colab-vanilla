# 고객 인도 및 운영 환경 구축 가이드

이 문서는 개발된 시스템을 최종 고객에게 인도하고, 고객이 자신의 구글 드라이브(GDF) 환경에서 시스템을 초기화 및 운용하는 방법을 설명합니다.

## 1. 초기 환경 구축 방법 (고객 직접 수행)

고객은 본인의 구글 계정에서 아래 순서에 따라 환경을 구축합니다.

### 1단계: 관리용 구글 시트 복사
1.  인도받은 구글 시트 주소에 접속합니다.
2.  상단 메뉴의 **[파일] > [사본 만들기]**를 클릭하여 본인의 드라이브로 가져옵니다.
    - *주의: 이때 Apps Script(소스 코드)도 함께 복사됩니다.*

### 2단계: 서버 정보 설정
1.  복사된 시트의 **'settings' 탭**으로 이동합니다.
2.  `vercelUrl` 항목에 본인이 배포한 웹사이트 주소를 입력합니다 (예: `https://my-exam.vercel.app`).
3.  `revalidateSecret` 등을 확인합니다 (기본값 권장).

### 3단계: 시스템 마스터 빌드 실행
1.  시트 상단 메뉴 **[💎 [모의고사 통합 관리]] > [🚀 1. 시스템 초기 빌드]**를 클릭합니다.
2.  권한 승인 팝업이 뜨면 모두 **[허용]**합니다.
3.  자동으로 고객 드라이브 내에 `Solutions` 폴더(해설지)와 `Forms` 폴더(설문지)가 생성되며 초기 데이터가 세팅됩니다.

---

## 2. 모의고사 추가 및 관리 방법

### 신규 모의고사 추가 순서
1.  **해설지 생성**: `Solutions` 폴더 안에 구글 문서를 새로 만듭니다.
2.  **설문지 생성**: `Forms` 폴더 안에 응답을 받을 구글 폼을 만듭니다.
3.  **실습 파일(Colab) 준비**: 다음 두 가지 방식 중 하나를 선택합니다.
    - **방식 A (구글 드라이브)**: `.ipynb` 파일을 구글 드라이브에 업로드하고 [링크가 있는 모든 사용자에게 공개] 후 주소 복사.
    - **방식 B (웹 서버)**: 개발자를 통해 웹사이트의 `public/colab/` 폴더에 파일을 추가하고 배포 (자동 주소 생성).
4.  **시트 등록**: `exams` 탭의 아래 행에 새 시험 정보를 입력합니다.
    - `solutionDoc`과 `formName`은 폴더에 만든 파일 이름과 정확히 일치해야 합니다.
4.  **동기화**: 상단 메뉴의 **[🔄 2. 드롭다운 목록 파일 동기화]**를 눌러 파일 ID를 자동으로 매칭합니다.
5.  **서버 반영**: **[⚡ 3. 수정한 내용 서버 즉시 반영]**을 눌러 웹사이트에 노출시킵니다.

## 3. 자주 묻는 질문 (FAQ)

**Q. 시트를 복사했는데 메뉴가 나타나지 않습니다.**
> **A.** 시트를 처음 열면 스크립트를 로드하는 데 5~10초 정도 시간이 걸립니다. 잠시 기다려도 안 나오면 페이지를 새로고침 하세요.

**Q. 웹사이트에 수정한 내용이 안 뜹니다.**
> **A.** 구글 시트는 DB가 아니므로 수정 후 반드시 상단 메뉴의 [⚡ 3. 수정한 내용 서버 즉시 반영] 버튼을 눌러야 서버 캐시가 갱신되어 반영됩니다.

**Q. 인증 코드를 더 늘리고 싶습니다.**
> **A.** `auth_codes` 탭의 아래 빈칸에 원하는 코드를 추가하고 `isActive`를 `TRUE`로 체크하면 즉시 사용 가능합니다.

**Q. Colab 주소는 어떤 형식을 써야 하나요?**
> **A.** 구글 드라이브 파일을 쓴다면 `https://colab.research.google.com/drive/파일ID` 형식을, 웹 서버에 올렸다면 `https://도메인/colab/파일명.ipynb` 형식을 사용하세요.

**Q. 고객이 Apps Script 코드를 수정해야 하나요?**
> **A.** 아니요. 모든 관리는 시트의 탭(exams, settings, auth_codes) 안에서 이루어지도록 설계되었습니다. 스크립트 수정은 고도화 시에만 필요합니다.

**Q. 버튼 실행 시 '안전하지 않은 앱'이라는 경고가 뜹니다.**
> **A.** 구글이 공식 승인하기 전의 개인 스크립트에서 발생하는 정상적인 경고입니다. [고객 정보] -> [...로 이동]을 눌러 권한을 허용하시면 됩니다.
