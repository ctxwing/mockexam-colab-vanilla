## 2026-04-03

### 문제점 1: auth.js에서 redirectIfAuthenticated 함수 미정의 오류

**증상:**
- `auth.js:139`에서 `Uncaught ReferenceError: redirectIfAuthenticated is not defined` 오류 발생
- 인증 페이지 로드 시 콘솔 에러로 인해 이후 스크립트 실행 중단

**원인:**
- `auth.js`의 DOMContentLoaded 이벤트 핸들러에서 `redirectIfAuthenticated()` 함수를 호출했으나, 해당 함수가 어디에도 정의되지 않음

**해결:**
- `common.js`에 `redirectIfAuthenticated()` 함수 추가
- localStorage 또는 sessionStorage에서 `authToken`을 확인하고, 존재 시 `lobby.html`로 리다이렉트
- 인증된 경우 `true` 반환하여 호출자가 추가 로직 실행을 중단할 수 있도록 함

**수정 파일:**
- `prj_source/js/common.js` - `redirectIfAuthenticated()` 함수 추가

### 문제점 2: start.sh 실행 시 포트 중복 점유로 인한 서버 기동 실패

**증상:**
- `./start.sh` 재실행 시 `OSError: [Errno 98] Address already in use` 오류 발생
- 기존 프로세스가 제대로 종료되지 않아 새 서버 기동 불가

**원인:**
- 초기 프로세스 정리 로직이 `pgrep` 패턴 매칭만 사용하여 모든 경우의 중복 프로세스를 잡지 못함
- PID 파일이 삭제되거나 불일치할 경우 포트 점유 해제 불가

**해결:**
- `lsof -ti:$PORT`를 사용하여 포트를 점유한 모든 프로세스를 강제 종료 (`kill -9`)
- PID 파일 기반 정리 + 포트 기반 정리 이중화

**수정 파일:**
- `start.sh` - 포트 점유 프로세스 강제 종료 로직 추가

### 문제점 3: OTP 입력 - 영문+숫자 조합 미지원 및 보안 이슈

**증상:**
- "AICE2025"와 같은 영문+숫자 조합 OTP 코드 입력 시 형식 검증 오류 발생
- `"요청한 형식과 일치시키세요"` 메시지 표시
- 붙여넣기 기능이 정상 동작하지 않음
- `otp-list.json`에 원본 코드가 평문으로 저장되어 GitHub Pages 배포 시 노출 위험

**원인:**
- HTML input 태그에 `pattern="[0-9]*"`로 숫자만 허용하도록 설정됨
- `inputmode="numeric"`으로 숫자 키패드만 표시
- JavaScript에서 숫자만 허용하는 정규식 `/^\d*$/` 사용
- 붙여넣기 시 숫자만 추출하는 `replace(/\D/g, '')` 로직
- OTP 코드가 평문 JSON으로 저장되어 프론트엔드에서 노출

**해결:**
1. **입력 필드 수정**:
   - `pattern="[A-Za-z0-9]"`로 영문+숫자 허용
   - `inputmode="text"`로 일반 키보드 표시
   - JavaScript에서 자동 대문자 변환 및 특수문자 필터링

2. **붙여넣기 기능 개선**:
   - 첫 번째 칸에 붙여넣어도 항상 첫 번째 칸부터 순차 분배
   - 하이픈 자동 제거 및 대문자 변환

3. **보안 강화 (SHA-256 해시)**:
   - 원본 OTP 코드는 `otp-codes.json`에 저장 (`.gitignore`로 관리)
   - 빌드 시점에 SHA-256 해시로 변환하여 `otp-list.json` 생성
   - 프론트엔드에서는 입력된 코드를 해시하여 비교
   - GitHub Pages 배포 시 해시만 노출되어 원본 코드 보호

**수정 파일:**
- `prj_source/pages/auth.html` - OTP input pattern 및 inputmode 수정
- `prj_source/js/auth.js` - 영문+숫자 허용, SHA-256 해시 검증 로직 추가
- `prj_source/data/otp-codes.json` - 원본 OTP 코드 (신규, .gitignore 관리)
- `prj_source/data/otp-list.json` - SHA-256 해시 목록 (빌드 시 자동 생성)
- `scripts/generate-otp-hashes.js` - OTP → 해시 변환 스크립트 (신규)
- `prj_source/.gitignore` - `data/otp-codes.json` 추가

**사용 방법:**
```bash
# OTP 코드 추가/변경 시
# 1. prj_source/data/otp-codes.json에 원본 코드 수정
# 2. 빌드 스크립트 실행
node scripts/generate-otp-hashes.js
```

**보안 참고:**
- 단방향 해시(SHA-256) 사용으로 역공개 어려움
- 완전한 보안은 서버사이드 검증이 필요하나, GitHub Pages 환경에서 실용적인 대안
- 하이브리드 배포 시 실제 인증 서버와 연동 권장

### 문제점 4: lobby.js에서 requireAuth 함수 미정의 오류

**증상:**
- `lobby.js:8`에서 `Uncaught ReferenceError: requireAuth is not defined` 오류 발생
- 로비 페이지 로드 시 인증 확인 로직 실행 실패

**원인:**
- `lobby.js`의 `initLobby()` 함수에서 `requireAuth()` 함수를 호출했으나, 해당 함수가 어디에도 정의되지 않음
- `common.js`에 `redirectIfAuthenticated()`는 있으나 `requireAuth()`는 누락

**해결:**
1. **`requireAuth()` 함수 추가** (`common.js`):
   - `isAuthenticated()` 함수로 인증 상태 확인
   - 인증되지 않은 경우 `auth.html`로 리다이렉트
   - 인증된 경우 `true` 반환하여 계속 진행

2. **Storage 키 일관성 문제 해결**:
   - 기존 `redirectIfAuthenticated()`가 직접 `localStorage.getItem('authToken')` 사용 (camelCase)
   - `storage.js`에서는 `'auth_token'` 키 사용 (snake_case)
   - `redirectIfAuthenticated()`를 `isAuthenticated()` 사용하도록 수정하여 키 불일치 문제 해결

**수정 파일:**
- `prj_source/js/common.js` - `requireAuth()` 함수 추가, `redirectIfAuthenticated()` 키 일관성 수정

### 개선 1: V1 실제 데이터 반영 및 시험 데이터 CRUD 관리 시스템 구축

**배경:**
- V2 프로젝트의 `exams.json`이 mock 데이터로만 구성됨
- V1 프로젝트(`project_source_V1`)의 실제 데이터를 반영 필요
- 관리자가 시험 데이터를 쉽게 CRUD할 수 있는 시스템 부재
- OTP 인증코드 관리 방식과 유사한 접근 필요

**구현 내용:**

1. **V1 데이터 반영**:
   - AICE 실기 모의고사 1~2회 데이터 이관
   - 빅데이터분석기사 실기 모의고사 1~2회 데이터 이관
   - 실제 Colab URL, 해설 파일 경로, 강의 링크 포함

2. **이중 파일 구조 (OTP 방식 참고)**:
   - `exams-source.json`: 관리자 편집용 (gitignore, 미커밋)
   - `exams.json`: 배포용 (빌드 스크립트로 생성)

3. **관리자 페이지 (`admin.html`)**:
   - 웹 UI 기반 CRUD 작업
   - 시험 추가/편집/삭제 기능
   - 파일 다운로드/업로드 기능
   - 활성/비활성 상태 관리

4. **빌드 스크립트 (`build-exams.js`)**:
   - 필수 필드 검증
   - URL 유효성 검사
   - ID 중복 확인
   - `isActive: false` 항목 필터링

**수정 파일:**
- `prj_source/data/exams.json` - V1 실제 데이터로 업데이트
- `prj_source/data/exams-source.json` - 관리자 편집용 소스 (신규)
- `scripts/build-exams.js` - 빌드 스크립트 (신규)
- `prj_source/pages/admin.html` - 관리자 페이지 (신규)
- `prj_source/css/admin.css` - 관리자 페이지 스타일 (신규)
- `prj_source/js/admin.js` - 관리자 페이지 로직 (신규)
- `prj_source/.gitignore` - `data/exams-source.json` 추가

**사용 방법:**
```bash
# 1. 관리자 페이지 접속
http://localhost:53500/pages/admin.html

# 2. 시험 데이터 추가/편집 후 [저장하기] 버튼 클릭
# 3. 다운로드된 exams-source.json을 prj_source/data/에 저장

# 4. 빌드 스크립트 실행
node scripts/build-exams.js

# 5. Git 커밋
git add prj_source/data/exams.json
git commit -m "chore: 시험 데이터 갱신"
git push
```
