# 외부 서비스 설정 가이드

## 1. Kakao JavaScript SDK 설정

### 1.1 Kakao Developers 앱 등록

1. [Kakao Developers](https://developers.kakao.com/)에 접속합니다.
2. 우측 상단 "로그인" 클릭 후 카카오 계정으로 로그인
3. "내 애플리케이션" → "애플리케이션 추가하기" 클릭
4. 앱 이름 입력 (예: "AICE 모의고사 플랫폼")
5. 생성된 앱의 "앱 키" 복사 (JavaScript 키)

### 1.2 플랫폼 설정

1. 앱 설정 → "플랫폼" 메뉴
2. "Website 플랫폼 추가" 클릭
3. 도메인 입력 (예: `https://yourdomain.github.io`)
4. 저장

### 1.3 코드 적용

`prj_source/pages/mobile-intro.html`에 다음 코드를 추가합니다:

```html
<!-- Kakao SDK 로드 -->
<script src="https://developers.kakao.com/sdk/js/kakao.min.js"></script>
<script>
  // 앱 키 설정 (실제 키로 수정)
  Kakao.init('YOUR_JAVASCRIPT_KEY');
  
  // 카카오톡 공유 함수
  function shareToKakao() {
    Kakao.Share.sendDefault({
      objectType: 'feed',
      content: {
        title: 'AICE 데이터 분석 모의고사',
        description: 'PC 실습 환경에 최적화된 서비스입니다',
        imageUrl: 'https://yourdomain.github.io/assets/images/book-cover.png',
        link: {
          mobileWebUrl: window.location.href,
          webUrl: window.location.href
        }
      }
    });
  }
</script>
```

### 1.4 테스트용 앱 키 (개발용)

개발 단계에서는 다음 테스트용 키를 사용할 수 있습니다:
- **JavaScript 키**: `d1234567890abcdef1234567890abcdef` (예시)
- 실제 배포 전 반드시 실제 앱 키로 변경하세요.

## 2. Google Programmable Search Engine 설정

### 2.1 검색 엔진 생성

1. [Google Programmable Search Engine](https://programmablesearchengine.google.com/)에 접속
2. "시작하기" 클릭
3. 검색 엔진 이름 입력 (예: "AICE 라이브러리 문서 검색")
4. 검색할 사이트 추가:
   - `numpy.org/doc/*`
   - `pandas.pydata.org/docs/*`
   - `scikit-learn.org/stable/*`
   - `matplotlib.org/stable/*`
5. "만들기" 클릭

### 2.2 검색 엔진 ID 확인

1. 생성된 검색 엔진 선택
2. "개요" 페이지에서 "검색 엔진 ID" 복사
3. 형식: `0123456789abcdef:ghijklmnop`

### 2.3 코드 적용

`prj_source/pages/lobby.html`에 다음 코드를 추가합니다:

```html
<!-- Google Programmable Search Element -->
<script async src="https://cse.google.com/cse.js?cx=YOUR_SEARCH_ENGINE_ID"></script>
<div class="gcse-search"></div>
```

### 2.4 테스트용 검색 엔진 ID

개발 단계에서는 다음을 사용할 수 있습니다:
- **검색 엔진 ID**: `0123456789abcdef:ghijklmnop` (예시)
- 실제 배포 전 반드시 실제 ID로 변경하세요.

## 3. GitHub Pages 설정

### 3.1 Repository 생성

```bash
# 로컬에서 GitHub 초기화
cd prj_source
git init
git add .
git commit -m "feat: 초기 프로젝트 구조 생성"

# GitHub Repository 생성 후 연결
git remote add origin https://github.com/yourusername/aice-mockexam.git
git branch -M main
git push -u origin main
```

### 3.2 GitHub Pages 활성화

1. GitHub Repository 접속
2. Settings → Pages 메뉴
3. Source 설정:
   - Branch: `main`
   - Folder: `/docs` 또는 `/root`
4. 저장

### 3.3 커스텀 도메인 설정 (선택)

1. Pages 설정 → Custom domain 입력
2. CNAME 파일 생성 (`prj_source/CNAME`):
   ```
   yourdomain.com
   ```
3. DNS 설정:
   - A 레코드: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - CNAME 레코드: `yourusername.github.io`

### 3.4 배포 자동화 (GitHub Actions)

`.github/workflows/deploy.yml` 생성:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './prj_source'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```
