# 시험 데이터 관리 운영 가이드

## 개요

본 문서는 AICE 모의고사 V2 프로젝트의 시험 데이터 CRUD 관리 방법을 설명합니다. GitHub Pages 배포 환경에서 운영되며, OTP 인증코드 관리 방식과 동일한 원리로 동작합니다.

---

## 1. 시스템 구조

### 1.1 파일 구조

```
mockexam-colab-vanilla/
├── prj_source/
│   ├── data/
│   │   ├── exams-source.json    # 🔴 관리자 편집용 (gitignore, 미커밋)
│   │   └── exams.json            # 🟢 배포용 (자동 생성, 커밋됨)
│   └── pages/
│       └── admin.html            # 관리자 페이지
├── scripts/
│   └── build-exams.js            # 빌드 스크립트
└── .gitignore                    # exams-source.json 제외 설정
```

### 1.2 데이터 흐름

```
┌─────────────────┐

└─────────────────┘
```

## 2. 데이터 스키마

### 2.1 exams-source.json 구조

```json
{
    "_comment": "관리자 전용 시험 데이터 소스 파일. 수정 후 'node scripts/build-exams.js' 실행하여 exams.json을 생성하세요.",
    "exams": [
        {
            "id": 1,
            "title": "AICE 실기 모의고사 1회",
            "description": "내비게이션 주행 데이터 기반 목적지 도착 시간 예측",
            "colabUrl": "https://colab.research.google.com/...",
            "solutionUrl": "../solutions/aice-01.md",
            "lectureUrl": "https://velog.io/...",
            "isActive": true
        }
    ]
}
```

### 2.2 필드 설명

| 필드 | 타입 | 필수 | 설명 |
|-----|------|-----|------|
| `id` | number | ✅ | 고유 ID (양의 정수, 중복 불가) |
| `title` | string | ✅ | 시험 제목 |
| `description` | string | ✅ | 시험 설명 |
| `colabUrl` | string | ✅ | Google Colab 실습 URL |
| `solutionUrl` | string | ✅ | 정답 및 해설 파일 경로 |
| `lectureUrl` | string | ❌ | 강의/참고자료 URL |
| `isActive` | boolean | ❌ | 활성 상태 (기본: true) |

## 3. 관리자 페이지 사용법

### 3.1 접근 방법

```
http://localhost:53500/pages/admin.html
```

> **주의**: 관리자 페이지는 별도의 인증이 없습니다. 실제 운영 시에는 인증 로직이 필요합니다.

### 3.2 CRUD 작업

#### 3.2.1 시험 추가

1. **[+ 시험 추가]** 버튼 클릭
2. 필드 입력:
   - 제목, 설명 (필수)
   - Google Colab URL (필수)
   - 정답 및 해설 경로 (필수)
   - 강의/참고자료 URL (선택)
   - 활성 상태 체크
3. **[저장]** 버튼 클릭

#### 3.2.2 시험 편집

1. 목록에서 **[편집]** 버튼 클릭
2. 내용 수정
3. **[저장]** 버튼 클릭

#### 3.2.3 시험 삭제

1. 목록에서 **[삭제]** 버튼 클릭
2. 확인 대화상자에서 **[확인]** 클릭

#### 3.2.4 파일 업로드

1. **[📤 업로드]** 버튼 클릭
2. `exams-source.json` 파일 선택
3. 기존 데이터를 덮어씁니다

### 3.3 데이터 저장

1. **[💾 저장하기]** 버튼 클릭
2. `exams-source.json` 파일이 다운로드됩니다
3. 다운로드한 파일을 `prj_source/data/` 폴더에 저장합니다

## 4. 빌드 및 배포 절차

### 4.1 빌드 스크립트 실행

```bash
# 프로젝트 루트에서
node scripts/build-exams.js
```

### 4.2 빌드 스크립트 기능

- **검증**: 필수 필드 확인, URL 유효성 검사, ID 중복 확인
- **필터링**: `isActive: false` 항목 제외
- **정렬**: ID 순서대로 정렬
- **변환**: `isActive` 필드 제거 (프론트엔드에서 불필요)

### 4.3 빌드 성공 시 출력

```
📚 시험 데이터 빌드 시작...

📖 4개 시험 데이터 로드 완료

✅ [1] AICE 실기 모의고사 1회
✅ [2] AICE 실기 모의고사 2회
✅ [3] 빅데이터분석기사 실기 모의고사 1회
✅ [4] 빅데이터분석기사 실기 모의고사 2회

✅ 빌드 완료!
📁 4개 시험 데이터가 생성되었습니다.
📄 저장 위치: /home/ctxwing/docker-ctx/lancer/mockexam-colab-vanilla/prj_source/data/exams.json

💡 다음으로 커밋하세요: git add prj_source/data/exams.json && git commit -m "chore: 시험 데이터 갱신"
```

### 4.4 Git 커밋

```bash
git add prj_source/data/exams.json
git commit -m "chore: 시험 데이터 갱신"
git push
```

> **중요**: `exams-source.json`은 `.gitignore`에 포함되어 커밋되지 않습니다.

## 6. 검증 규칙

빌드 스크립트는 다음 규칙으로 데이터를 검증합니다:

### 6.1 필수 필드 검사

- `id`, `title`, `description`, `colabUrl`, `solutionUrl` 필수

### 6.2 ID 유효성 검사

- ID는 양의 정수여야 함 (1, 2, 3, ...)
- ID 중복 불가

### 6.3 URL 유효성 검사

- `colabUrl`: 유효한 URL 형식 필수
- `lectureUrl`: 유효한 URL 형식 (있는 경우)

### 6.4 검증 실패 시

```
❌ [1] 제목 없음
   - 필수 필드 'title' 누락
   - 'colabUrl'이 유효한 URL이 아닙니다

❌ 검증 오류가 있어 빌드를 중단합니다.
```

## 7. 관리 시나리오

### 7.1 신규 시험 추가

1. 관리자 페이지 접속
2. `[+ 시험 추가]` 버튼 클릭
3. 시험 정보 입력
4. `[💾 저장하기]`로 다운로드
5. `exams-source.json`을 `prj_source/data/`에 저장
6. `node scripts/build-exams.js` 실행
7. `git add/commit/push`

### 7.2 기존 시험 수정

1. 관리자 페이지 접속
2. 해당 시험의 `[편집]` 버튼 클릭
3. 내용 수정 후 `[저장]`
4. `[💾 저장하기]`로 다운로드
5. `exams-source.json`을 `prj_source/data/`에 덮어쓰기
6. `node scripts/build-exams.js` 실행
7. `git add/commit/push`

### 7.3 시험 비활성화

1. 관리자 페이지에서 해당 시험 편집
2. `활성 상태` 체크박스 해제
3. 저장 후 빌드
4. `isActive: false`인 항목은 `exams.json`에서 제외됨

### 7.4 일괄 가져오기

기존 `exams-source.json` 파일을 관리자 페이지에서 업로드:

1. `[📤 업로드]` 버튼 클릭
2. 파일 선택
3. 데이터 로드 후 편집 가능

## 8. 보안 고려사항

### 8.1 현재 보안 모델

- 관리자 페이지: **인증 없음** (로컬 개발용)
- `exams-source.json`: **gitignore**로 원격 저장소 미노출
- `exams.json`: **공개** (GitHub Pages로 배포)

### 8.2 운영 시 보안 강화 방안

| 방안 | 설명 | 구현 난이도 |
|-----|------|-----------|
| OTP 인증 연동 | 기존 OTP 시스템 활용 | 쉬움 |
| 관리자 전용 페이지 | 별도 도메인/경로 | 쉬움 |
| IP 제한 | GitHub Actions에서만 빌드 허용 | 중간 |
| 서버사이드 API | Vercel/Netlify Functions 활용 | 어려움 |

## 9. 문제 해결

### 9.1 빌드 실패

**증상**: `❌ 검증 오류가 있어 빌드를 중단합니다.`

**해결**:
1. 오류 메시지 확인
2. `exams-source.json`에서 해당 항목 수정
3. 빌드 재실행

### 9.2 ID 중복

**증상**: `❌ 중복된 ID가 있습니다: 1, 3`

**해결**:
1. 중복된 ID 중 하나를 변경
2. 관리자 페이지에서 자동 ID 할당 기능 사용

### 9.3 URL 오류

**증상**: `❌ 'colabUrl'이 유효한 URL이 아닙니다`

**해결**:
1. URL 형식 확인 (https:// 포함)
2. 브라우저에서 URL 테스트

## 11. 정리

### 장점

1. **간편한 관리**: 웹 UI로 CRUD 작업 가능
2. **안전한 배포**: 소스 파일이 Git에 노출되지 않음
3. **검증 자동화**: 빌드 시 데이터 무결성 확인
4. **GitHub Pages 친화적**: 정적 호스팅에 최적화

### 단점

1. **관리자 인증 미구현**: 별도 인증 로직 필요
2. **빌드 단계 필요**: 소스 → 결과 변환 과정 필요
3. **실시간 반영 불가**: 빌드 후 커밋해야 반영됨
