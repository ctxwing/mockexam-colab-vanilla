## 1. 개요

GDF(Google Drive Folder) 방식 구현을 위해 사전에 준비해야 할 항목입니다.
아래 순서대로 진행해 주세요.

## 3. 폴더 구조 생성

### 3.1 메인 폴더 생성

1. Google Drive 접속 (drive.google.com)
2. "새로 만들기" > "폴더"
3. 폴더명: `모의고사_관리` (또는 원하는 이름)
4. 생성된 폴더 열기

### 3.2 하위 폴더 생성

메인 폴더 안에서 다음 폴더들 생성:

| 폴더명 | 용도 |
|--------|------|
| `Solutions` | 해설 문서 (Google Docs) 저장 |
| `Forms` | 제출 폼 (Google Forms) 저장 |
| `Responses` | 폼 응답 시트 저장 (자동 생성) |
| `Colabs` | Colab 노트북 바로가기 (선택) |

### 3.3 최종 폴더 구조

```
📁 모의고사_관리/
├── 📁 Solutions/
├── 📁 Forms/
├── 📁 Responses/
└── 📁 Colabs/
```

---

## 4. 관리 시트 생성

### 4.1 스프레드시트 생성

1. 메인 폴더(`모의고사_관리`) 안에서
2. "새로 만들기" > "Google 스프레드시트"
3. 파일명: `문항관리`

### 4.2 시트 탭 추가

스프레드시트 하단에서 탭 추가 (+):

| 탭 이름 | 용도 |
|---------|------|
| exams | 모의고사 목록 |
| auth_codes | 인증 코드 목록 |
| settings | 설정값 |

### 4.3 exams 시트 헤더 (1행)

다음 헤더를 1행에 입력:

```
id | title | description | category | solutionDoc | solutionDocId | solutionUrl | formName | formUrl | colabUrl | dataUrl | order | isActive
```

### 4.4 auth_codes 시트 헤더 (1행)

```
code | isActive | note
```

### 4.5 settings 시트 (키-값)

| A열 (키) | B열 (값) |
|----------|----------|
| solutionsFolderId | (나중에 입력) |
| formsFolderId | (나중에 입력) |
| lastUpdated | (나중에 GAS가 자동 입력) |

---

## 5. 폴더 ID 확인 방법

각 폴더의 ID를 settings 시트에 입력해야 합니다.

### 5.1 폴더 ID 확인

1. 해당 폴더 열기 (예: Solutions)
2. 브라우저 주소창 확인
3. URL 형식: `https://drive.google.com/drive/folders/XXXXXXXXXX`
4. `folders/` 뒤의 문자열이 **폴더 ID**

### 5.2 settings 시트에 입력

| 키 | 값 예시 |
|----|---------|
| solutionsFolderId | 1AbCdEfGhIjKlMnOpQrStUvWxYz |
| formsFolderId | 2BcDeFgHiJkLmNoPqRsTuVwXyZa |

## 6. 인증 코드 준비

### 6.1 코드 생성

총 20개의 인증 코드 준비 (도서별 고유 코드)

**형식 권장:**
- 16자리 영숫자
- 4자리씩 하이픈 구분
- 예: `A1B2-C3D4-E5F6-G7H8`

### 6.2 코드 생성 도구 (선택)

온라인 난수 생성기 사용 가능:
- random.org
- 또는 개발자에게 생성 요청

### 6.3 auth_codes 시트에 입력

| code | isActive | note |
|------|----------|------|
| A1B2-C3D4-E5F6-G7H8 | TRUE | 초판 1쇄 |
| B2C3-D4E5-F6G7-H8I9 | TRUE | 초판 1쇄 |
| ... | ... | ... |

## 7. 콘텐츠 준비

### 7.1 Colab 노트북 URL

각 모의고사별 Colab 노트북 URL 목록 준비

| 모의고사 | Colab URL |
|----------|-----------|
| 빅분기 1회 | https://colab.research.google.com/drive/... |
| 빅분기 2회 | https://colab.research.google.com/drive/... |
| AICE 1회 | https://colab.research.google.com/drive/... |
| AICE 2회 | https://colab.research.google.com/drive/... |

### 7.2 실습 데이터 URL

pandas에서 직접 로드 가능한 CSV URL

| 데이터셋 | URL |
|----------|-----|
| 빅분기 1회 데이터 | https://raw.githubusercontent.com/... |
| ... | ... |

### 7.3 해설 문서

**옵션 A: 개발자가 기존 MD 파일을 Docs로 변환 (권장)**
- 개발자가 마이그레이션 수행
- 고객 작업 불필요

**옵션 B: 고객이 직접 작성**
- Solutions 폴더에 Google Docs로 작성
- 공유 설정: "링크가 있는 모든 사용자" + "뷰어"

### 7.4 제출 폼 (Google Forms)

각 모의고사별 제출 폼 생성

**개발자 지원 가능:**
- 폼 템플릿 제공
- 또는 개발자가 직접 생성

## 8. 공유 설정

### 8.1 메인 폴더 공유

1. 메인 폴더 우클릭 > "공유"
2. "일반 액세스" > "링크가 있는 모든 사용자" (또는 "제한됨" 유지)
3. 개발자 이메일 추가 > "편집자" 권한

### 8.2 Solutions 폴더 내 문서 공유

**각 해설 문서마다:**
1. 문서 열기 > "공유" 버튼
2. "일반 액세스" > "링크가 있는 모든 사용자"
3. 권한: "뷰어"
