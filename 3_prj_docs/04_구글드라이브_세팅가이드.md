##  자동 접근 제한 알림 (필독)

**"최초 1회 스크립트 실행"**은 반드시 직접 수행 필요.

대신, 번거로운 폴더 생성, 파일 만들기, 폼 제작 등은 **제공된 스크립트(`setup` 함수)가 자동으로 수행**하도록 코딩되어 있음.
아래 **4단계**만 진행해 주시면 나머지는 자동으로 완료됨.

---

## 1. 메인 폴더 접속

1. 생성한 공유 폴더에 접속.
   - 링크: [prj_qr_mockexam_colab_202512](https://drive.google.com/drive/folders/1cgpL8rudRmSPRuU7HkuIypwTV0B2bLj9?usp=drive_link)
2. **주의:** 반드시 이 폴더 **안에서** 다음 단계를 진행 필요.

---

## 2. '문항관리' 시트 생성 (1분 소요)

1. 위 폴더 안에서 **"새로 만들기" > "Google 스프레드시트"** 클릭
2. 파일 이름을 `문항관리`로 변경
3. 생성된 시트 열기

---

## 3. 매직 스크립트 붙여넣기 (2분 소요)

1. 시트 상단 메뉴에서 **[확장 프로그램] > [Apps Script]** 클릭
2. 코드 에디터 화면이 뜨면 (Code.gs), **기존 내용을 모두 지웁니다.**
3. 아래 링크(또는 제가 드린 파일)에 있는 **코드를 전체 복사(`Ctrl+A`, `Ctrl+C`)**하여 붙여넣습니다(`Ctrl+V`).
    - 초기 세팅용: `3_prj_docs/MockExamInitialSetup.gs`
    - 상시 운용용: `3_prj_docs/MockExamOperation.gs`
    - (최초 시트 생성 후 초기 세팅 시에는 `InitialSetup`을, 그 이후 상시 메뉴 사용 시에는 `Operation` 코드로 교체 권장)
4. 저장 (`Ctrl+S`, 프로젝트 이름은 `MockExamBackend` 등 아무거나)

---

## 4. 자동 세팅 실행 (가장 중요!)

이 단계에서 모든 폴더와 파일이 자동으로 생성됨.

1. 코드 에디터 상단 툴바에서 함수 선택 박스를 클릭 > **`setup`** 선택
2. **[실행]** 버튼 클릭
3. **권한 승인 팝업**이 뜨면:
   - [권한 검토] > 계정 선택 > **"Google hasn’t verified this app" 화면이 나옴**
   - 좌측 하단 **[Advanced(고급)]** 클릭
   - 맨 아래 **[Go to MockExamBackend (unsafe) / (으)로 이동(안전하지 않음)]** 클릭
   - **[Allow(허용)]** 클릭
4. 실행 완료 메시지("세팅 및 서식 적용 완료!")가 뜰 때까지 대기.

✅ **이 과정이 끝나면 자동으로 완료되는 작업들:**
- `Solutions`, `Forms`, `Responses`, `Colabs` 폴더 생성
- `exams`, `auth_codes`, `settings` 시트 생성 및 헤더 설정
- 해설 문서(Google Docs) 4종 자동 생성 (서식 적용됨)
- 구글 폼 4종 자동 생성 및 연동

---

## 5. 웹 앱 배포 (마지막 단계)

Next.js 웹사이트와 연동 작업.

1. Apps Script 화면 우측 상단 **[배포] > [새 배포]**
2. 톱니바퀴 아이콘 > **[웹 앱]**
3. 설정 (반드시 아래와 같이!)
   - 설명: `v1`
   - 다음 사용자로 실행: **나(Me)** (필수)
   - 액세스 권한: **모든 사용자(Anyone)** (필수)
4. **[배포]** 클릭
5. **[액세스 승인]** 버튼 클릭
6. **"Google hasn’t verified this app" (확인되지 않은 앱) 화면이 나올 경우:**
   - 좌측 하단 **[Advanced (고급)]** 클릭
   - 맨 아래 **[Go to MockExamBackend (unsafe) / (으)로 이동(안전하지 않음)]** 클릭
   - 구글 계정 로그인 요청에 로그인 진행
   - **[Allow (허용)]** 클릭
8. **상시 반영을 위한 추가 설정 (중요)**
   - 생성된 웹 앱 URL을 복사하여 `.env.local`의 `NEXT_PUBLIC_GAS_API_URL`에 입력합니다.
   - 스프레드시트의 `settings` 시트로 이동합니다.
   - **vercelUrl**: 배포된 웹사이트 주소를 입력합니다 (예: `https://your-site.vercel.app`).
   - **revalidateSecret**: `ctx-admin-secret` 또는 `.env.local`에 설정한 시크릿 값을 입력합니다.
   - 이 설정이 완료되어야 스프레드시트 메뉴 3번(Purge)이 정상 작동합니다.

## 요약
자동화가 불가능한 초기 설정 작업 영역(로그인, 최초 생성)만 진행 후, setup script가 자동으로 나머지 작업을 처리함.
위 **1 ~ 5단계만** 수동 작업 진행함
