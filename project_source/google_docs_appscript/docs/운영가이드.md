# 시스템 운영 및 관리 가이드 (v2.0 - GAS 통합 버전)
작성일자: 2025-12-23

본 문서는 Google Apps Script(GAS)를 활용한 교육용 가상 모의고사 시스템의 유지보수 및 효율적인 운영을 위한 가이드라인입니다.

## 1. 데이터 관리 구조
본 시스템은 **Google 스프레드시트**를 데이터베이스(DB)로, **Next.js**를 웹 프론트엔드로 사용합니다. 모든 콘텐츠 수정은 스프레드시트에서 이루어집니다.

### 1.1 콘텐츠 관리 위치
- **모의고사 목록**: `문항관리` 스프레드시트 > `exams` 시트
- **인증 코드 관리**: `문항관리` 스프레드시트 > `auth_codes` 시트
- **시스템 설정**: `문항관리` 스프레드시트 > `settings` 시트

## 2. 실시간 데이터 동기화 (ISR 최적화)
웹사이트 로딩 속도를 극대화하기 위해 서버 사이드 캐싱(ISR)이 적용되어 있습니다.

### 2.1 자동 갱신
- 서버는 1시간(3600초)마다 최신 데이터를 구글 시트에서 자동으로 가져옵니다.

### 2.2 즉시 갱신 (Push/Purge)
관리자가 시트 내용을 수정한 후 즉시 반영하고 싶을 때는 다음을 실행합니다.
- **방법**: 스프레드시트 메뉴 `💎 [모의고사 통합 관리]` > `⚡ 3. 수정한 내용 서버 즉시 반영 (Purge)` 클릭
- **원리**: Vercel 서버의 기존 캐시를 강제로 삭제(Purge)하고 시트의 최신 데이터를 즉시 불러옵니다.

## 3. 인증 및 보안 운영
### 3.1 로그아웃 기능
- 웹사이트 상단의 인증 배지 우측 **로그아웃 아이콘**을 클릭하면 즉시 인증 정보가 삭제됩니다.
- 로그아웃 상태에서는 대시보드 및 모든 하위 페이지 접근이 차단되며 로그인 페이지로 리다이렉트됩니다.

### 3.2 미들웨어 보호
- `middleware.ts` 파일이 `/dashboard` 하위의 모든 요청을 감시하며, 비인증 사용자의 접근을 차단합니다.

## 4. 문제 및 해설지 추가 가이드
- **해설지**: Google Docs로 작성 후 `Solutions` 폴더에 업로드합니다.
- **제출 폼**: Google Forms로 작성 후 `Forms` 폴더에 업로드합니다.
- **목록 동기화**: 파일을 업로드한 후 스프레드시트 메뉴에서 `🔄 2. 드롭다운 목록 파일 동기화`를 수행해야 `exams` 시트에서 선택이 가능합니다.

## 5. 긴급 대응 프로세스
1. **잘못된 문제 배포 시**: `exams` 시트에서 해당 행의 `isActive` 체크 해제 -> `3. 서버 즉시 반영` 실행.
2. **인증 코드 대량 유출 시**: `auth_codes` 시트에서 해당 코드 비활성화 -> `3. 서버 즉시 반영` 실행.
